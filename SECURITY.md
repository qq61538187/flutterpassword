## 报告安全问题

**请勿在公开的 GitHub Issue 中报告安全问题。**

如果你发现了安全漏洞，请通过以下方式报告：

### 报告方式

1. **发送邮件**：发送到项目维护者的邮箱（请在 GitHub 个人资料中查找）
2. **创建私有安全建议**：在 GitHub 上创建 [Security Advisory](https://github.com/qq61538187/flutterpassword/security/advisories/new)

### 报告内容

请在报告中包含以下信息：

- **漏洞描述**：详细描述发现的安全问题
- **影响范围**：说明该漏洞可能造成的影响
- **复现步骤**：提供详细的复现步骤（如可能）
- **建议修复方案**：如果你有修复建议，欢迎提供
- **你的联系方式**：以便我们与你联系获取更多信息

### 响应时间

我们承诺：

- **24 小时内**：确认收到你的报告
- **7 天内**：评估漏洞的严重程度
- **30 天内**：发布修复补丁（根据严重程度调整）

## 安全最佳实践

### 对于用户

1. **使用强主密码**：
   - 至少 12 位字符
   - 包含大小写字母、数字和特殊字符
   - 不要使用常见密码或个人信息

2. **定期备份**：
   - 使用导出功能定期备份数据
   - 将备份文件存储在安全的位置

3. **保护主密码**：
   - 不要与他人分享主密码
   - 不要将主密码存储在未加密的地方
   - 考虑使用密码提示功能帮助记忆

4. **启用自动锁定**：
   - 在设置中启用自动锁定功能
   - 设置合理的锁定时间

5. **保持应用更新**：
   - 及时更新到最新版本
   - 关注安全更新通知

### 对于开发者

1. **代码审查**：
   - 所有涉及加密、认证的代码都需要仔细审查
   - 遵循安全编码最佳实践

2. **依赖管理**：
   - 定期更新依赖包
   - 关注依赖包的安全公告

3. **测试**：
   - 包含安全相关的测试用例
   - 进行渗透测试（如可能）

## 已知安全问题

### 当前版本的限制

1. **本地存储**：
   - 当前版本仅支持本地存储，不支持云同步
   - 数据丢失风险：如果设备损坏，数据可能无法恢复

2. **密码提示**：
   - 密码提示以明文形式存储（仅用于帮助回忆，不包含密码信息）

3. **内存安全**：
   - 主密码在内存中可能短暂存在，应用关闭后会清除

## 安全审计

### 加密方案

- **主密码哈希**：PBKDF2-SHA256，6000 次迭代
- **数据加密**：AES-256
- **密钥派生**：PBKDF2
- **盐值**：每个条目使用独立的随机盐值
- **IV**：每个加密操作使用独立的随机 IV

### 安全建议

1. **迭代次数**：
   - 当前 PBKDF2 迭代次数为 6000
   - 这是性能和安全性的平衡点
   - 未来可能会根据硬件性能调整

2. **密钥管理**：
   - 主密码不会存储在设备上
   - 加密密钥从主密码派生，不直接存储

3. **数据完整性**：
   - 建议定期验证数据完整性
   - 使用导出功能进行备份
